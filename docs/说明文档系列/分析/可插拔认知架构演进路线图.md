# AIRI 自驱系统演进路线图 — 可插拔认知架构

**作者** AI 产品架构专家 · **日期** 2026-03-09

---

## 核心设计原则：认知供应商接口 (CPI)

引入 **Cognitive Provider Interface (CPI)** — 一层薄协议层，让桥接层只依赖协议，不依赖具体实现。

```
AIRI 前端 (消费者)
    │  HTTP/WS
    ▼
CPI 协议层 (ICognitiveProvider ABC)
    │  impl
    ▼
可插拔实现: NarrativeEngine / Mock / 纯LLM / 未来第三方
```

---

## 四阶段演进计划

### Phase 1: 基础修复 (1-2 天)

> 目标：让现有代码正确运行，不增加新功能

| 任务                     | 说明                                                 |
| ------------------------ | ---------------------------------------------------- |
| Manager → Singleton      | 所有 `_try_get_*` 改用 `get_xxx_manager()` singleton |
| 接入 Layer 0 感知        | `get_time_manager()` + 天气 + 位置                   |
| 增加记忆容量             | 记忆 `[:50]` → `[:200]`, 条数 3 → 5                  |
| prompt 需求数值→自然语言 | "体力值: 850" → "精力充沛"                           |

### Phase 2: CPI 协议层引入 (2-3 天)

> 目标：建立插拔边界，让桥接层不再直接 import 核心模块

```
api/cpi/
├── interface.py           ← ICognitiveProvider ABC
├── narrative_engine.py    ← NarrativeEngine 实现 (singleton 聚合)
├── mock_provider.py       ← 测试/降级用空实现
└── __init__.py             ← factory: get_cognitive_provider()
```

### Phase 3: 认知深度模块逐个接通 (1-2 周)

```
[1] 感知聚合器          合并 time/weather/position/social
[2] 语义记忆检索        替代 recent_history → embedding 检索
[3] 意识流真实触发      调用 process_consciousness_stream()
[4] prompt 结构化       权重分层 + 负面指令 + MBTI 全系列
[5] RGES 闭环验证       经验 → 反思 → 目标演化 → 人格变化
[6] L1-L3 记忆升级      对话中提取关键信息 → 升级到高层记忆
```

### Phase 4: 自主意识流闭环 (后续)

角色不仅在被动对话时展现认知，还能主动产生想法和行为。

## 避免耦合的 6 条铁律

| #   | 规则                                  | 违反示例                             | 正确做法                            |
| --- | ------------------------------------- | ------------------------------------ | ----------------------------------- |
| 1   | **桥接层只依赖 CPI，不 import core/** | `from core.xxx import Xxx`           | `from api.cpi import get_provider`  |
| 2   | **每个认知模块可独立禁用**            | 记忆模块崩溃 → 整个桥接 500          | try/except + 静态降级               |
| 3   | **配置驱动而非代码驱动**              | `if mbti == "ENTJ"` 硬编码           | JSON 配置 → 模板渲染                |
| 4   | **前端不知道后端用什么认知引擎**      | `fetch('/api/narrative-engine/...')` | `fetch('/api/airi/...')` 统一端点   |
| 5   | **LLM Provider 可替换**               | 硬编码 Kimi API                      | `unified_llm_config.py` 多 provider |
| 6   | **测试不依赖运行中的服务**            | 集成测试必须启动 NarrativeEngine     | MockCognitiveProvider 支持单测      |

---

## 执行纪律 — 三个必须遵守的原则

1. **Singleton 纪律** — 认知模块必须是单例，不能每次 new
2. **异步意识流** — UCS 处理在后台运行，不阻塞对话路径
3. **可降级设计** — 每个模块崩溃时自动降级到静态数据，而非全系统崩溃
