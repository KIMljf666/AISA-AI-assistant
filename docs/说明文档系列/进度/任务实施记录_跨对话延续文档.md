## 一、项目目标

将 NarrativeEngine 的自驱 AI 系统（9层意识流、L1-L4记忆、四层心理、RGES反思演化、通用感知）集成到 AIRI VTuber 角色"赛希斯"(Saihisis, ENTJ) 上。

**LLM 双模型方案**:
- **日常对话**: `moonshot-v1-auto` (Moonshot API, ~2-5s)
- **深度推理**: `kimi-k2.5` (temperature=1, ~8-30s, 含 reasoning_content)

**TTS 方案**:
- **引擎**: edge-tts (微软神经网络语音, 免费, 零延迟)
- **声线**: XiaoxuanNeural (晓萱, 优雅知性 — 赛希斯专属)
- **端点**: `POST /v1/audio/speech` (OpenAI 兼容)

## 二、当前架构

```
AIRI 前端 (Vue/Pinia, port 5200, strictPort)
  ├── chat.ts → fetchEnrichedPrompt() + submitDialogueFeedback()
  ├── narrative-bridge.ts → HTTP/WS 通信 + 双模型 + config CRUD + 🆕 proactive_dialogue
  ├── Stage.vue → TTS 播放 + 音量驱动 fallback 口型同步
  ├── Model.vue → Live2D 渲染 + 空闲口型抑制(说话时)
  └── CognitiveBubble.vue v2.0 → 4 Tab: 📡实时流 / 🪪身份卡 / 🎭角色编辑 / ⚙️设置
       │  🆕 因果连线 + 记忆溯源展开 + 记忆涟漪动画 + 主动发言高亮
       │  HTTP GET/POST/PUT + WebSocket
       ▼
airi_bridge.py (FastAPI 桥接层, port 5555)
  ├── /character-state/{id}/system-prompt → L0→L1→L2→L3→L5→L6 事件广播
  │     🆕 L3 记忆广播含 search_method + similarity scores + 完整记忆列表
  ├── /dialogue-feedback → L7 reasoning + L9 反思事件
  ├── /character-config/{id} → GET/PUT 角色配置 CRUD
  ├── /model-config → 双模型查询/切换
  ├── /v1/audio/speech → Edge-TTS 语音合成 (OpenAI 兼容)
  ├── /v1/audio/voices → 8种中文声线列表
  └── _get_provider() → CPI 协议层
       │
       ▼
api/cpi/ (认知供应商接口) — 完全可插拔
  ├── interface.py → ICognitiveProvider ABC (13 方法)
  ├── narrative_engine.py → NE 实现 (深度合并 + JSON persist)
  │     🆕 VectorMemoryIndex (sentence-transformers, 384维, cosine相似度)
  │     🆕 recall_memories() → 向量语义检索优先 + 关键词 fallback
  │     🆕 store_memory() → 同步到向量索引
  │     🆕 _extract_key_facts_llm() → Moonshot API 结构化提取
  └── real_world_perception.py → 真实世界感知
```
