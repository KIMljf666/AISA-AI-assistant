## 三、P1 阶段计划: 意识流管道深度验证

### 3.1 LLM 关键信息提取测试 (`test_llm_key_fact_extraction.py`)

**目标**: 验证 L2/L3 记忆层的 LLM 驱动提取能力

**LLM API 优先级**:
1. 🥇 Google Gemini (`gemini-2.0-flash`) — 通过 OpenAI 兼容接口调用
2. 🥈 阿里云百炼 (`qwen3.5-plus`)
3. 🥉 Moonshot (`moonshot-v1-auto`) — 现有实现

**测试用例**:

| # | 测试场景 | 输入 | 期望提取 |
|---|---------|------|---------|
| 1 | 姓名提取 | "我叫小明" | `用户姓名: 小明` |
| 2 | 偏好提取 | "我最喜欢吃披萨和拉面" | `用户偏好: 披萨`, `用户偏好: 拉面` |
| 3 | 关系提取 | "我妹妹叫小红" | `用户关系: 妹妹小红` |
| 4 | 情绪提取 | "今天考试没过，很沮丧" | `用户情绪: 沮丧` |
| 5 | 无信息 | "今天天气真好" | `[]` (空列表) |
| 6 | 复合信息 | "我叫张三，喜欢蓝色，讨厌下雨" | ≥3条提取 |
| 7 | 降级验证 | API 不可用时 | 降级到规则引擎 |

**验收标准**: 6/7 场景通过，提取准确率 ≥ 80%

### 3.2 意识流管道完整性测试 (`test_consciousness_pipeline.py`)

**目标**: 验证 UCS 9层因果链 → GLM 目标管理 → ERE 反思 的完整管道

**测试用例**:

| # | 测试场景 | 验证点 |
|---|---------|--------|
| 1 | UCS 意识流生成 | 给定刺激，UCS 应生成包含感知→认知→决策的分层响应 |
| 2 | UCS 状态持久化 | 同一角色两次调用 `get_consciousness()` 应反映状态延续 |
| 3 | GLM 目标生命周期 | 创建目标→执行→完成/放弃 的完整周期 |
| 4 | GLM 目标优先级排序 | 多目标场景下优先级排序正确 |
| 5 | ERE 经验提交 | submit_experience 后 ERE 应产生反思洞察 |
| 6 | ERE RGES 触发 | 含 reasoning_content 的经验应触发 RGES |
| 7 | 管道联动 | 刺激→UCS处理→GLM目标调整→ERE反思 全链路 |
| 8 | 主动对话触发 | consciousness_loop 产生 proactive_dialogue 事件 |

**验收标准**: 核心链路 (1,2,5,7) 必须通过

### 3.3 记忆系统深度验证 (`test_memory_deep.py`)

**目标**: 验证向量语义检索 vs 关键词检索的效果差异

**测试用例**:

| # | 测试场景 | 验证点 |
|---|---------|--------|
| 1 | 精确匹配 | "小明" → 包含"小明"的记忆 |
| 2 | 语义匹配 | "学生叫什么" → 包含"小明"的记忆 (语义关联) |
| 3 | 跨轮累积 | 5轮对话后，recall 应返回跨轮信息 |
| 4 | L2 升级验证 | key_fact 提取后存入 L2，优先级高于 L4 |
| 5 | 记忆衰减 | 旧记忆分数低于新记忆 |
| 6 | 大量记忆性能 | 100+ 记忆下检索耗时 < 500ms |

**验收标准**: 语义匹配 (2) 召回率 ≥ 60%，性能 (6) 达标
