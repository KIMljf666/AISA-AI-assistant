## 二、CPI 接口 (Character Provider Interface)

CPI 是意识流系统的**唯一对外接口**, MVP 只需调用这些方法即可获得完整的 AI 角色能力。

### 核心方法清单

| # | 方法 | 文件位置 | 功能 |
|---|------|---------|------|
| 1 | `get_character_config(char_id)` | `narrative_engine.py:326` | 获取角色完整配置 (人格/背景/四层心理) |
| 2 | `recall_memories(char_id, query, limit)` | `narrative_engine.py:474` | **语义记忆检索** (向量+遗忘+强化) |
| 3 | `store_memory(char_id, content, layer)` | `narrative_engine.py:524` | 存储记忆 (自动同步向量索引) |
| 4 | `get_active_goals(char_id)` | `narrative_engine.py:574` | 获取当前目标 |
| 5 | `submit_experience(char_id, experience)` | `narrative_engine.py:644` | 提交游戏经历 → 触发 RGES 反思 |
| 6 | `get_enriched_system_prompt(...)` | `airi_bridge.py:690` | **生成完整 System Prompt** (核心!) |
| 7 | `_retrieve_relevant_moments(char_id, query)` | `airi_bridge.py:62` | LifeMoment 重要度检索 |
| 8 | `_build_proactive_directives(char_id)` | `airi_bridge.py:140` | 生成主动行为指令 |
| 9 | `_build_state_observation(char_id)` | `airi_bridge.py:210` | 四层心理动态观察 |
| 10 | `_get_recent_user_messages(char_id)` | `airi_bridge.py:264` | 跨轮对话历史 |

### 最小接入示例

```python
from narrative_engine.api.cpi.narrative_engine import NarrativeEngineProvider

provider = NarrativeEngineProvider()

# 1. 获取角色配置
config = provider.get_character_config("saihisis")

# 2. 检索相关记忆 (语义向量检索 + 遗忘曲线 + 重复强化)
memories = provider.recall_memories("saihisis", query="紫府令旗", limit=5)
# → [{"content": "...", "score": 0.76, "access_count": 3, "search_method": "vector"}]

# 3. 生成完整 System Prompt (包含四层心理/记忆/感知/行为指令)
prompt = await get_enriched_system_prompt("saihisis", query="你好",
    game_context={"highest_floor": 9, "total_deaths": 3})
# → 1200+ chars 结构化 prompt

# 4. 对话后存储记忆 (自动同步向量索引)
provider.store_memory("saihisis", "用户询问了关于紫府令旗的记忆", layer="L4")

# 5. 提交游戏经历 → 触发 RGES 反思
provider.submit_experience("saihisis", {
    "type": "battle", "result": "victory", "floor": 9
})
```

## 三、记忆系统架构

### 3.1 四层记忆模型 (类人设计)

| 层 | 人类对应 | 实现方式 | 存储位置 |
|----|---------|---------|---------|
| **感觉记忆** | 毫秒级 | LLM context window | 运行时 |
| **工作记忆** | L2 Key Facts | DeepSeek API 提取 | `{char}_dialogue.jsonl` |
| **短期记忆** | L4 对话 | JSONL + 向量索引 (384d) | `{char}_dialogue.jsonl` + `{char}.pkl` |
| **长期记忆** | LifeMoment | 结构化 JSONL | `{char}/moments.jsonl` |

### 3.2 类人记忆机制

| 机制 | 公式/原理 | 效果 |
|------|----------|------|
| **遗忘曲线** | `score × e^(-0.02 × hours)` | 1天保留62%, 7天仅3.5% |
| **重复强化** | `score × (1 + 0.1 × access_count)` | 被引用越多越容易回忆 |
| **睡眠整合** | `memory_consolidation.py` | 高频记忆聚类 → L2 长期摘要 |
| **重要度分层** | death=3.0 > legendary=2.5 > boss=2.0 | 死亡/传说装备优先召回 |

### 3.3 记忆检索流程

```
用户输入 → encode(query) → 384维向量
                ↓
    VectorMemoryIndex.search(query, top_k=5)
                ↓
    ┌─ cosine 相似度 ─── 基础得分
    ├─ 遗忘曲线衰减 ──── × decay
    ├─ 重复强化加权 ──── × (1+0.1×count)
    └─ 排序 → top-5 ──→ 注入 prompt
```

### 3.4 记忆涟漪 (前端可视化)

```
recall_memories() → broadcast("memory_recall", {memories, top_score})
        ↓ WebSocket
narrative-bridge.ts → CognitiveBubble.vue
        ↓
fuchsia 波纹动画 (0.8s) + 记忆溯源列表展开
```

## 四、认知系统架构

### 4.1 9层因果链 (UCS 意识流处理器)

```
L1 核心价值 → L2 身份认同 → L3 记忆层 → L4 动机层
     ↓              ↓            ↓           ↓
L5 目标层 → L6 行为规划 → L7 执行层 → L8 反馈层
     ↓
L9 反思层 (RGES)
```

### 4.2 RGES 反思引擎

```
submit_experience(battle_result)
        ↓
    quality_score < 0.5?
        ↓ Yes
    _trigger_dialogue_reflection()
        ↓
    生成反思: "[反思] 下次回应应更具体"
        ↓
    存入 L3 记忆层 → 下一轮 prompt 自动注入
```

### 4.3 Prompt 注入层 (get_enriched_system_prompt)

完整的 System Prompt 由以下模块**自动组装**:

| 序号 | 模块 | 示例内容 |
|------|------|---------|
| 1 | 角色设定 | "你是赛希斯, 深渊的引导者..." |
| 2 | 大五人格 | "O:0.85 C:0.70 E:0.75 A:0.80 N:0.30" |
| 3 | 四层心理 | 核心价值/身份/动机/状态 |
| 4 | 环境感知 | 日期/时段/节日/天气 |
| 5 | 身体状态 | needs → 自然语言 |
| 6 | 当前目标 | "引导外来者/研究种子秘密" |
| 7 | 相关记忆 | 向量检索 top-5 |
| 8 | LifeMoment | 重要度排序 top-3 |
| 9 | 内心想法 | 意识流处理器输出 |
| 10 | **状态观察** | "外来者已深入第9层, 经历3次死亡" |
| 11 | **最近对话** | "用户说过: 紫府令旗..." |
| 12 | 反思 | RGES 近期反思 |
| 13 | **主动行为指令** | "提及观察到的经历 / 分享建议" |

## 六、MVP 接入指南

### 6.1 文件结构

```
narrative_engine/
├── api/
│   ├── airi_bridge.py          # 🔑 核心桥接 (prompt生成/感知/行为)
│   └── cpi/
│       ├── narrative_engine.py  # 🔑 CPI 实现 (15+方法)
│       └── memory_consolidation.py  # 睡眠整合
├── data/
│   ├── airi_memory/             # 记忆存储
│   │   ├── {char}_dialogue.jsonl
│   │   └── {char}/moments.jsonl
│   └── vector_index/            # 向量索引
│       └── {char}.pkl
└── tests/integration/
    └── simulate_player.py       # A/B 测试 + 双重评分
```

### 6.2 接入步骤

**Step 1**: 创建角色配置

```yaml
# data/characters/{char_id}.yml
id: my_npc
name: "角色名"
personality:
  mbti: "ENTP"
  big_five: {openness: 0.8, conscientiousness: 0.6, ...}
  four_layers:
    core: "核心价值观"
    self: "身份认同"
    motivation: "当前动机"
    state: "情绪状态"
```

**Step 2**: 创建 LifeMoment 数据

```jsonl
{"event_type": "death", "original_intention": "探索第9层", "significance_hint": 0.9, "actions_taken": [{"result_summary": "被Boss击败"}]}
{"event_type": "legendary_drop", "original_intention": "获得紫府令旗", "significance_hint": 0.95, "actions_taken": [{"result_summary": "装备提升战力"}]}
```

**Step 3**: 调用 CPI

```python
# 在你的对话处理函数中:
prompt = await get_enriched_system_prompt(
    character_id="my_npc",
    query=user_message,
    game_context=current_game_state  # {highest_floor, total_deaths, best_equipment, ...}
)
# 将 prompt 作为 system message 发送给 LLM
```
